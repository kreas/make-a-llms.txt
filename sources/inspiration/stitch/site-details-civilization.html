# Stripe API Documentation Overview

This document provides a high-level summary of the Stripe API structure and key endpoints, optimized for LLM context inclusion.

## Core Resources

- **Charges**: To charge a credit or a debit card, you create a `Charge` object.
  - `POST /v1/charges`
  - `GET /v1/charges/:id`

- **Customers**: `Customer` objects allow you to perform recurring charges and track multiple charges that are associated with the same customer.
  - `POST /v1/customers`
  - `GET /v1/customers/:id`

- **PaymentIntents**: The PaymentIntents API is the recommended way to integrate Stripe.
  - `POST /v1/payment_intents`
  - `POST /v1/payment_intents/:id/confirm`

## Authentication
Authenticate your account when using the API by including your secret API key in the request.
